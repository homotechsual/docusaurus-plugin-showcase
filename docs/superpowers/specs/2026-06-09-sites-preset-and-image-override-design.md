# Sites Preset and Local Image Override

**Date:** 2026-06-09
**Reference:** `https://github.com/facebook/docusaurus/blob/main/website/src/data/users.tsx`

---

## Goal

Add a `sitesPreset` that mirrors the docusaurus.io/showcase format (sites built with Docusaurus, not plugins), and add a local image override mechanism: images placed alongside YAML files in `dataDir` are served automatically, with a fallback chain of local file â†’ `preview` URL â†’ screenshot service.

---

## Decisions

- **`statuses` becomes non-mandatory** â€” remove the "must have â‰Ą 1 status" validation; allow `statuses: {}`. The status filter section in `ShowcaseFilters` is hidden when `Object.keys(options.statuses).length === 0`. The `sitesPreset` explicitly passes `statuses: {}` since the Docusaurus site showcase has no maintenance statuses.
- **Bundler-agnostic image serving** â€” instead of generating a `showcase-data.json` file, `contentLoaded` generates `showcase-data.js` â€” a valid ES module. Items with a discovered local image emit `preview: require('/abs/path/image.png')`. The bundler (webpack or Rspack) handles the `require()` through its asset pipeline without any plugin-level bundler configuration. No `CopyWebpackPlugin`, no `devServer.static`.
- **Image discovery by matching basename** â€” given `mysite.yaml`, the loader checks for `mysite.png`, `mysite.jpg`, `mysite.jpeg`, `mysite.webp`, `mysite.gif`, `mysite.svg` in that priority order. First match wins.
- **`title` accepted as an alias for `name`** â€” the Docusaurus source uses `title` not `name`. The loader normalises `title â†’ name` before validation when `name` is absent. The sites schema allows both fields; the generic showcase schema is unchanged.
- **No new TypeScript type export for the internal image path** â€” `_localImagePath` is carried as an extra property on `ShowcaseItem` (which already has `[key: string]: unknown`) and stripped when generating the module. It is never part of the exported `ShowcaseItem` contract.
- **Sites schema is strict** â€” `additionalProperties: false`, fields limited to `id`, `name`, `title`, `description`, `website`, `source`, `preview`, `tags`. No `author`, `status`, `npmPackages`, or `minimumVersion`.

### 2. Optional statuses â€” `src/plugin.ts`

**`validateOptions`:** Remove the block that throws when `statuses` is empty:

```ts

  throw new Error('[docusaurus-plugin-showcase] The `statuses` option must define at least one status.')
}
```

Normalise `statuses` to `{}` when absent:

```ts

```

**`contentLoaded`:** Generate a JS module instead of JSON:

```ts
const showcaseDataPath = await createData(
  'showcase-data.js',
  generateShowcaseModule(content),
)
```

The `generateShowcaseModule` helper (defined in `plugin.ts`, not exported) serialises items to JS, substituting `require('/abs/path')` for items with `_localImagePath`:

```ts
function generateShowcaseModule(data: ShowcasePageData): string {
  const serializeItem = (item: ShowcaseItem): string => {
    const { _localImagePath, preview, ...rest } = item as ShowcaseItem & { _localImagePath?: string }
    const previewJs = _localImagePath
      ? `require(${JSON.stringify(_localImagePath)})`
      : JSON.stringify(preview ?? null)
    const restContent = JSON.stringify(rest).slice(1, -1) // inner content without braces
    return restContent
      ? `{"preview":${previewJs},${restContent}}`
      : `{"preview":${previewJs}}`
  }
  const itemsJs = data.items.map(serializeItem).join(',')
  return `// @generated by @homotechsual/docusaurus-plugin-showcase\nexport default {"items":[${itemsJs}],"options":${JSON.stringify(data.options)}};\n`
}
```

### 7. Image co-location + `title â†’ name` â€” `src/loaders/yaml.ts`

After `yaml.load()` and before schema validation, apply two normalisations:

```ts
const raw = yaml.load(readFileSync(filePath, 'utf-8'))

if (typeof raw !== 'object' || raw === null) { /* existing warn + continue */ }

// 1. title â†’ name alias
const item = raw as Record<string, unknown>
if (!item.name && item.title) {
  item.name = item.title
  delete item.title
}

// 2. Co-located image detection
const { basename, extname } = await import('node:path')
const base = basename(filePath, extname(filePath))  // e.g. "algolia"
const imageExts = ['.png', '.jpg', '.jpeg', '.webp', '.gif', '.svg']
for (const ext of imageExts) {
  const imagePath = join(dirname(filePath), `${base}${ext}`)
  if (existsSync(imagePath)) {
    item._localImagePath = imagePath
    break
  }
}

if (validate && !validate(item)) { /* existing warn + continue */ }

items.push(item as ShowcaseItem)
```

Update the `node:path` import at the top of the loader to include the newly needed names:

```ts
import { resolve, join, dirname, basename, extname } from 'node:path'
```

`existsSync` is already imported from `node:fs`.
